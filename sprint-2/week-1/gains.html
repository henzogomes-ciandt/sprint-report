Metodologia e premissasMethodology & assumptions

  • Horas com IA = OriginalEstimate do ADO — dimensionado já contando com o agente. É o campo preenchido de forma consistente; onde há CompletedWork (real), o valor foi menor que a estimativa (o agente superou a própria estimativa), então o ganho aqui está subestimado, não inflado.Hours with AI = ADO OriginalEstimate — sized already assuming the agent. It's the consistently-filled field; where CompletedWork (actual) exists it was lower than the estimate (the agent beat its own estimate), so the gain here is understated, not inflated.
  • Estimativa manual (sem IA) = tempo de um desenvolvedor sênior fazendo a task sem IA, ciclo completo (entender o requisito e o código → implementar → testar → abrir PR → ciclos de review). Estimada task a task a partir do escopo real do que foi entregue, não de uma fórmula. O agente comprime justamente os custos fixos humanos (leitura de código, boilerplate, testes, iterações de review); por isso o multiplicador é maior em tasks de scaffolding/porte e menor onde a lógica/decisão de design domina. Multiplicadores deliberadamente conservadores (2,0–3,0×; 2,5× no total).Manual estimate (no AI) = a senior developer doing the task without AI, full cycle (understand the requirement and the code → implement → test → open the PR → review iterations). Estimated task by task from the actual scope of what was delivered, not from a formula. The agent compresses exactly the fixed human costs (reading code, boilerplate, tests, review iterations); that's why the multiplier is higher on scaffolding/port tasks and lower where logic/design judgment dominates. Deliberately conservative multipliers (2.0–3.0×; 2.5× overall).
  • Escopo: apenas as 24 tasks concluídas na semana, com estimativa agentic. Excluídas as tasks ainda abertas/em review, a criação do repositório (812739, DevOps, sem estimativa) e o seeding de dados de mock (812808, enabler feito sem o agente).Scope: only the 24 tasks completed this week that carry an agentic estimate.
