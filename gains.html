Metodologia e premissasMethodology & assumptions

  • Horas com IA = OriginalEstimate do ADO — dimensionado já contando com o agente. É o campo preenchido de forma consistente; onde há CompletedWork (real), o valor foi menor que a estimativa (o agente superou a própria estimativa), então o ganho aqui está subestimado, não inflado.Hours with AI = ADO OriginalEstimate — sized already assuming the agent. It's the consistently-filled field; where CompletedWork (actual) exists it was lower than the estimate (the agent beat its own estimate), so the gain here is understated, not inflated.
  • Estimativa manual (sem IA) = tempo de um desenvolvedor sênior fazendo a task sem IA, ciclo completo (entender o requisito e o código → implementar → testar → abrir PR → ciclos de review). Estimada task a task a partir do escopo real do que foi entregue, não de uma fórmula. O agente comprime justamente os custos fixos humanos (leitura de código, boilerplate, testes, iterações de review); por isso o multiplicador é maior em tasks de scaffolding/integração e menor onde a lógica/decisão de design domina. Multiplicadores deliberadamente conservadores (1,7–3,5×; 2,4× no total).Manual estimate (no AI) = a senior developer doing the task without AI, full cycle (understand the requirement and the code → implement → test → open the PR → review iterations). Estimated task by task from the actual scope of what was delivered, not from a formula. The agent compresses exactly the fixed human costs (reading code, boilerplate, tests, review iterations); that's why the multiplier is higher on scaffolding/integration tasks and lower where logic/design judgment dominates. Deliberately conservative multipliers (1.7–3.5×; 2.4× overall).
  • Escopo: apenas as 11 tasks entregues na semana. Excluídas a task de investigação fechada sem código (808158, no-op) e a ainda em aberto (808160, Strapi).Scope: only the 11 tasks delivered this week.
