Metodologia e premissasMethodology & assumptions

  • Horas com IA = OriginalEstimate do ADO — dimensionado já contando com o agente. É o campo preenchido de forma consistente; onde há CompletedWork (real), o valor foi menor que a estimativa (o agente superou a própria estimativa), então o ganho aqui está subestimado, não inflado.Hours with AI = ADO OriginalEstimate — sized already assuming the agent. It's the consistently-filled field; where CompletedWork (actual) exists it was lower than the estimate (the agent beat its own estimate), so the gain here is understated, not inflated.
  • Estimativa manual (sem IA) = tempo de um desenvolvedor sênior fazendo a task sem IA, ciclo completo (entender o requisito e o código → implementar → testar → abrir PR → ciclos de review). Estimada task a task a partir do escopo real do que foi entregue, não de uma fórmula. Multiplicadores deliberadamente conservadores (1,7–3,5×; 2,3× no total). O fator é maior na semana 1 (scaffolding/integração, onde o agente comprime custos fixos humanos) e menor na semana 2 (fiação de dados reais e decisões de contrato de API).Manual estimate (no AI) = a senior developer doing the task without AI, full cycle (understand the requirement and the code → implement → test → open the PR → review iterations). Estimated task by task from the actual scope delivered, not from a formula. Deliberately conservative multipliers (1.7–3.5×; 2.3× overall). The factor is higher in week 1 (scaffolding/integration, where the agent compresses fixed human costs) and lower in week 2 (real-data wiring and API-contract decisions).
