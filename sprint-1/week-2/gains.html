Metodologia e premissasMethodology & assumptions

  • Horas com IA = OriginalEstimate do ADO — dimensionado já contando com o agente. É o campo preenchido de forma consistente; onde há CompletedWork (real), o valor foi menor que a estimativa (o agente superou a própria estimativa), então o ganho aqui está subestimado, não inflado.Hours with AI = ADO OriginalEstimate — sized already assuming the agent. It's the consistently-filled field; where CompletedWork (actual) exists it was lower than the estimate (the agent beat its own estimate), so the gain here is understated, not inflated.
  • Estimativa manual (sem IA) = tempo de um desenvolvedor sênior fazendo a task sem IA, ciclo completo (entender o requisito e o código → implementar → testar → abrir PR → ciclos de review). Estimada task a task a partir do escopo real do que foi entregue, não de uma fórmula. Multiplicadores deliberadamente conservadores (2,0–2,5×; 2,1× no total) — semana de integração e fiação de dados reais, onde a lógica e as decisões de contrato de API pesam mais e o teto de ganho é naturalmente menor que na semana de scaffolding.Manual estimate (no AI) = a senior developer doing the task without AI, full cycle (understand the requirement and the code → implement → test → open the PR → review iterations). Estimated task by task from the actual scope delivered, not from a formula. Deliberately conservative multipliers (2.0–2.5×; 2.1× overall) — a week of integration and real-data wiring, where logic and API-contract decisions weigh more and the ceiling on gains is naturally lower than the scaffolding week.
